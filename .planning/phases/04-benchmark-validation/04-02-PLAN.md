Add a `verify` job to `.github/workflows/ci.yml` after the existing `source` job. The job:

1. Job definition:
   ```yaml
   verify:
     name: Verify native gem
     needs: build
     runs-on: ${{ matrix.os }}
     strategy:
       fail-fast: false
       matrix:
         include:
           - os: ubuntu-latest
             platform: x86_64-linux
           - os: macos-latest
             platform: arm64-darwin
   ```

2. Steps:
   - `uses: ruby/setup-ruby@v1` with `ruby-version: "3.4"`
   - `uses: actions/download-artifact@v4` with:
     - `name: bundle-artifact-${{ matrix.platform }}`
     - `path: pkg/`
   - Shell step named "Install and verify native gem":
     ```bash
     gem install pkg/*.gem
     ruby -e "require 'gte'; puts GTE::VERSION; puts 'Native gem loaded OK'"
     ```

The artifact names `bundle-artifact-x86_64-linux` and `bundle-artifact-arm64-darwin` match the existing `upload-artifact` step naming convention in the `build` job.

Do NOT modify the existing `build` or `source` jobs. Only append the new `verify` job.
