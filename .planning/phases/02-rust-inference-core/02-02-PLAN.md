    Function 1 — build_session:
    ```rust
    /// Create an ORT inference session from a local ONNX model file.
    /// Uses ORT v2 API: Session::builder()?.commit_from_file(path)
    /// NOT the v1 SessionBuilder::new() API (does not exist in ort 2.0.0-rc.9).
    pub fn build_session<P: AsRef<Path>>(model_path: P) -> crate::Result<Session> {
        let session = Session::builder()?
            .commit_from_file(model_path)?;
        Ok(session)
    }
    ```

    Function 2 — run_session:
    ```rust
    /// Run the ORT session on tokenized inputs and extract embeddings as Array2<f32>.
    ///
    /// CRITICAL: SessionOutputs borrows from Session. Extract Array2<f32> here before returning.
    /// Do NOT return SessionOutputs — it cannot outlive the session borrow.
    pub fn run_session(
        session: &Session,
        tokenized: &Tokenized,
        config: &ModelConfig,
    ) -> crate::Result<Array2<f32>> {
        // Build typed input HashMap — all inputs are i64 tensors
        let mut inputs: HashMap<&str, Value<ort::value::TensorValueType<i64>>> = HashMap::new();
        inputs.insert("input_ids", Value::from_array(tokenized.input_ids.view())?);
        inputs.insert("attention_mask", Value::from_array(tokenized.attn_masks.view())?);
        if let Some(ref type_ids) = tokenized.type_ids {
            inputs.insert("token_type_ids", Value::from_array(type_ids.view())?);
        }

        let outputs = session.run(inputs)?;

        // Extract the embedding tensor by name from session outputs
        let tensor_value = outputs
            .get(config.output_tensor)
            .ok_or_else(|| GteError::Inference(format!(
                "output tensor '{}' not found in model outputs",
                config.output_tensor
            )))?;

        let array = tensor_value.try_extract_tensor::<f32>()?;

        // Apply extraction mode: CLS token (rank-3 → rank-2) or Raw (rank-2 already)
        let embeddings: Array2<f32> = match config.mode {
            ExtractorMode::Token(idx) => {
                // Shape: [batch, seq, dim] — take token at idx (usually 0 = CLS)
                let slice = array.slice(ndarray::s![.., idx, ..]);
                slice.into_owned()
            }
            ExtractorMode::Raw => {
                // Shape: [batch, dim] — already pooled
                array.into_dimensionality::<Ix2>()?.into_owned()
            }
        };

        Ok(embeddings)
    }
    ```

    CRITICAL PITFALL (from RESEARCH.md Pitfall 1): Do NOT return `SessionOutputs` from this function. Extract `Array2<f32>` and return it — the owned array lives beyond `outputs` being dropped.

    CRITICAL PITFALL (from RESEARCH.md Pitfall 2): ort must have `features = ["ndarray"]` in Cargo.toml (done in Plan 01). If `Value::from_array` fails to compile, check the feature flag.

    CRITICAL PITFALL (from RESEARCH.md Pitfall 3): All input tensors are i64. The HashMap is typed to `Value<TensorValueType<i64>>`. Do NOT mix tensor types in the same map.

    CRITICAL PITFALL (from RESEARCH.md Pitfall 5): For Token(idx) mode, the tensor must be rank-3. E5's `last_hidden_state` is confirmed rank-3. If you hit an index-out-of-bounds or ndarray dimension error here, check that the model actually outputs rank-3 (not already pooled).
